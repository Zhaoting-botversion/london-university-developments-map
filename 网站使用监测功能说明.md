# 网站使用监测功能说明

## 目标

为 London University Developments Map 增加轻量使用监测，帮助后续判断：

- 每天是否有人访问网站
- 销售常用哪些学校筛选
- 常用哪些价格、户型、区域、状态筛选
- 哪些楼盘详情被打开得最多
- 多少项目被加入推荐选项
- 是否有人使用导出 PDF 功能
- 客户需求筛选功能是否被使用

## 已加入的前端埋点

目前 `index.html` 已支持以下事件：

- `map_page_view`：地图工具加载
- `school_selected` / `school_unselected`：学校多选切换
- `filter_price_toggled`：价格筛选
- `filter_bedroom_toggled`：户型筛选
- `filter_area_toggled`：区域筛选
- `filter_status_toggled`：收房状态筛选
- `client_requirements_applied`：客户需求筛选
- `development_detail_opened`：打开楼盘详情
- `development_selected` / `development_unselected`：加入或移出导出选项
- `selection_cleared`：清空选项
- `pdf_export_started`：点击导出 PDF
- `pdf_export_confirmed`：确认导语并进入打印导出

## GA4 监测

当前网站已接入 Google Analytics 4：

```js
G-3D05C5K1GC
```

GA4 会自动记录页面访问、设备、地区、来源等基础数据。上方自定义事件会通过 `gtag('event', ...)` 发送到 GA4，用于查看销售具体使用了哪些功能。

常用查看路径：

- Realtime：确认当前是否收到事件
- Reports > Engagement > Events：查看事件次数
- Admin > Events：查看事件定义
- Explore：按事件参数分析学校、筛选条件、导出项目等

## 隐私边界

当前设计不主动采集客户姓名、电话、微信等敏感信息。  
客户称呼和销售编辑的导语不会被上传。  
上传的是功能使用记录，例如筛选条件、选中的楼盘名称、学校范围、页面访问时间等。

## 后台推荐方案

当前网站是 GitHub Pages 静态网页，没有自带服务器。推荐先使用 Google Sheets + Apps Script 作为轻量后台：

1. 新建一个 Google Sheet，作为网站使用记录表。
2. 在 Google Sheet 中打开 `Extensions > Apps Script`。
3. 复制 `scripts/usage-analytics-google-apps-script.gs` 的内容到 Apps Script。
4. 将脚本中的 `SPREADSHEET_ID` 改成这份 Google Sheet 的 ID。
5. 点击 `Deploy > New deployment > Web app`。
6. 设置：
   - Execute as: Me
   - Who has access: Anyone
7. 部署后复制 Web App URL。
8. 在 `index.html` 的 analytics 配置处填入：

```js
window.LUDM_ANALYTICS_ENDPOINT = '你的 Apps Script Web App URL';
```

## 后台表格结构

Apps Script 会自动创建两个 Sheet：

- `Raw Events`：每一条用户行为记录
- `Daily Summary`：每日汇总

`Daily Summary` 可通过 Apps Script 的 `createDailySummary()` 生成，建议后续设置一个每天自动运行的 Trigger。

## 可优化方向

后续如果使用量增加，可以升级为：

- Google Analytics 4：更适合看访问量和流量来源
- Supabase：更适合自定义后台面板和复杂统计
- 公司内部接口：更适合纳入正式 CRM 或销售管理系统
